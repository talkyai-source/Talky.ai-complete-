## 4. DB hotfix ‚Äî "Add contact" bug ‚Äî DEPLOYED (`54e56df2`, CURRENT PROD)
**Symptom:** `Failed to create contact: invalid input for query argument $8: {} (expected str, got dict)` when adding a campaign contact.
**Root cause:** `postgres_adapter._execute_async` opens raw `asyncpg.connect()` connections (2 sites) that never registered the jsonb/json codec (only the pool did, via `init=`), while `_coerce_bind_value` deliberately passes dict/list values through to jsonb columns trusting that codec. On the ad-hoc path there was no codec ‚Üí asyncpg got a raw `{}` for `custom_fields` ($8). Broke **any** jsonb write through the adapter, not just contacts.
**Fix:** register the same codec on both ad-hoc connect sites. `FakeConn` test double gained `set_type_codec`; regression asserts the codec is registered on insert. Gate: **3306 passed**. Deployed + verified (healthz 200; first curl after restart returned `000` = pre-bind race, passed on retry).
